## 1. OpenAI API — Thin Python Wrapper Pattern

### How it works

The cleanest approach for direct OpenAI API usage is a **wrapper function** that
intercepts every `chat.completions.create()` call, queries MemoryWeb, and injects
relevant memories into the `system` message before forwarding.

### Implementation

```python
# memoryweb_openai.py
# Drop-in replacement for openai.chat.completions.create()

import json
import httpx
from openai import OpenAI

MW_URL = "http://localhost:8100"
_real_client = OpenAI()  # uses OPENAI_API_KEY from env

def chat_with_memory(messages: list[dict], model: str = "gpt-4o", **kwargs):
    """
    Drop-in for openai.chat.completions.create().
    Queries MemoryWeb with the last user message, prepends relevant memories
    to the system prompt, then forwards to OpenAI.
    """
    # Extract the latest user message for the memory query
    user_text = ""
    for m in reversed(messages):
        if m.get("role") == "user":
            user_text = m["content"] if isinstance(m["content"], str) else ""
            break

    memories_block = ""
    if user_text:
        try:
            r = httpx.post(
                f"{MW_URL}/api/search",
                json={"query": user_text, "limit": 5},
                timeout=3.0,
            )
            if r.status_code == 200:
                results = r.json().get("results", [])
                if results:
                    snippets = "\n".join(
                        f"- {item.get('content', '')[:400]}" for item in results
                    )
                    memories_block = (
                        f"\n\n[RELEVANT MEMORIES FROM MEMORYWEB]\n{snippets}\n"
                        "[END MEMORIES]\n"
                    )
        except Exception:
            pass  # MemoryWeb down → degrade gracefully, do not fail the call

    # Inject into existing system message or create one
    enriched = list(messages)
    if memories_block:
        sys_idx = next((i for i, m in enumerate(enriched) if m["role"] == "system"), None)
        if sys_idx is not None:
            enriched[sys_idx] = {
                "role": "system",
                "content": enriched[sys_idx]["content"] + memories_block,
            }
        else:
            enriched.insert(0, {"role": "system", "content": memories_block.strip()})

    return _real_client.chat.completions.create(
        model=model, messages=enriched, **kwargs
    )
```

### Usage

```python
from memoryweb_openai import chat_with_memory

response = chat_with_memory(
    messages=[{"role": "user", "content": "What was the AgentForge port number?"}]
)
print(response.choices[0].message.content)
```

### Notes
- Zero external dependencies beyond `openai` and `httpx` (both already installed).
- Timeout is 3 seconds — MemoryWeb failure never blocks the OpenAI call.
- Works with streaming: pass `stream=True` in kwargs, it forwards cleanly.

## 2. Cursor / Windsurf / IDE LLM Tools

### The landscape (as of 2026-03-30)

| Tool | Memory injection mechanism | Status |
|------|--------------------------|--------|
| **Cursor** | Rules files (`.cursorrules`), no MCP | Partial — static injection only |
| **Windsurf** | MCP client (Model Context Protocol) | Full dynamic injection possible |
| **Zed** | MCP client | Full dynamic injection possible |
| **Continue.dev** | MCP client + context providers | Full dynamic injection possible |

### Cursor (.cursorrules)

Cursor does not support MCP or dynamic context providers as of March 2026.
The best available approach is to pre-populate `.cursorrules` with static
memory summaries and refresh it periodically via a cron/watch script.

```
# .cursorrules (auto-generated by refresh script)
# MemoryWeb top memories as of [timestamp]
- [memory snippet 1]
- [memory snippet 2]
...
```

A refresh script would: `POST /api/search?query=general&limit=20` → write to `.cursorrules`.
This is a poor substitute for dynamic injection. Cursor is the weakest integration target.

### Windsurf, Zed, Continue.dev (MCP)

These tools are **MCP clients**. MemoryWeb can be exposed as an **MCP server**
(Model Context Protocol, defined at modelcontextprotocol.io). An MCP server
exposes "tools" and "resources" that the IDE calls automatically during context
assembly.

A MemoryWeb MCP server would expose:
- **Tool**: `search_memories(query: str, limit: int)` → returns relevant memories
- **Resource**: `memoryweb://recent` → last N memories as a readable resource

Build effort for the MCP adapter: ~4 hours (Python `mcp` SDK, single file).
Once deployed, Windsurf/Zed/Continue pick it up via `.mcp.json` config:

```json
{
  "mcpServers": {
    "memoryweb": {
      "command": "python",
      "args": ["-m", "memoryweb_mcp"],
      "cwd": "D:\\memory-web"
    }
  }
}
```

## 3. OpenAI-Compatible Proxy at Port 4141

### Architecture

```
Any LLM client (Cursor, scripts, anything)
         |
         | POST /v1/chat/completions
         v
  MemoryWeb Proxy (port 4141)
         |
         |-- POST /api/search → MemoryWeb (port 8100)
         |      ↳ inject memories into system message
         |
         | POST /v1/chat/completions (enriched)
         v
  OpenAI API (api.openai.com)
         |
         v
  Response passed back to client unchanged
```

The proxy is **OpenAI API-compatible**: any client that can set a base URL can
point at it. This is the most powerful integration — it requires zero changes to
individual tools beyond swapping the base URL.

### Full proxy implementation

```python
# memoryweb_proxy.py
# Run: python memoryweb_proxy.py
# Then set OPENAI_BASE_URL=http://localhost:4141/v1 in any client

import os
import json
import httpx
from fastapi import FastAPI, Request, Response
from fastapi.responses import StreamingResponse
import uvicorn

MW_URL = "http://localhost:8100"
OPENAI_URL = "https://api.openai.com"
OPENAI_KEY = os.environ.get("OPENAI_API_KEY", "")

app = FastAPI(title="MemoryWeb OpenAI Proxy", version="1.0.0")


def _extract_user_text(messages: list) -> str:
    for m in reversed(messages):
        if m.get("role") == "user":
            content = m.get("content", "")
            if isinstance(content, str):
                return content
            # Handle content arrays (vision messages)
            if isinstance(content, list):
                parts = [p.get("text", "") for p in content if p.get("type") == "text"]
                return " ".join(parts)
    return ""


def _inject_memories(messages: list, memories_block: str) -> list:
    enriched = list(messages)
    sys_idx = next((i for i, m in enumerate(enriched) if m.get("role") == "system"), None)
    if sys_idx is not None:
        enriched[sys_idx] = {
            **enriched[sys_idx],
            "content": enriched[sys_idx]["content"] + memories_block,
        }
    else:
        enriched.insert(0, {"role": "system", "content": memories_block.strip()})
    return enriched


async def _query_memoryweb(query: str) -> str:
    if not query:
        return ""
    try:
        async with httpx.AsyncClient(timeout=3.0) as client:
            r = await client.post(
                f"{MW_URL}/api/search",
                json={"query": query, "limit": 5},
            )
            if r.status_code == 200:
                results = r.json().get("results", [])
                if results:
                    snippets = "\n".join(
                        f"- {item.get('content', '')[:400]}" for item in results
                    )
                    return (
                        f"\n\n[RELEVANT MEMORIES FROM MEMORYWEB]\n{snippets}\n"
                        "[END MEMORIES]\n"
                    )
    except Exception:
        pass  # graceful degrade
    return ""


@app.post("/v1/chat/completions")
async def proxy_chat(request: Request):
    body = await request.json()
    messages = body.get("messages", [])
    stream = body.get("stream", False)

    # Query MemoryWeb with the latest user message
    user_text = _extract_user_text(messages)
    memories_block = await _query_memoryweb(user_text)

    # Inject memories if found
    if memories_block:
        body["messages"] = _inject_memories(messages, memories_block)

    # Forward to real OpenAI
    headers = {
        "Authorization": f"Bearer {OPENAI_KEY}",
        "Content-Type": "application/json",
    }
    # Pass through any extra auth headers from client
    if "authorization" in request.headers:
        headers["Authorization"] = request.headers["authorization"]

    if stream:
        # Streaming passthrough
        async def generate():
            async with httpx.AsyncClient(timeout=120.0) as client:
                async with client.stream(
                    "POST",
                    f"{OPENAI_URL}/v1/chat/completions",
                    json=body,
                    headers=headers,
                ) as resp:
                    async for chunk in resp.aiter_bytes():
                        yield chunk

        return StreamingResponse(generate(), media_type="text/event-stream")
    else:
        async with httpx.AsyncClient(timeout=120.0) as client:
            resp = await client.post(
                f"{OPENAI_URL}/v1/chat/completions",
                json=body,
                headers=headers,
            )
            return Response(
                content=resp.content,
                status_code=resp.status_code,
                media_type="application/json",
            )


@app.get("/v1/models")
async def proxy_models(request: Request):
    """Pass-through for model listing."""
    headers = {"Authorization": f"Bearer {OPENAI_KEY}"}
    if "authorization" in request.headers:
        headers["Authorization"] = request.headers["authorization"]
    async with httpx.AsyncClient(timeout=10.0) as client:
        resp = await client.get(f"{OPENAI_URL}/v1/models", headers=headers)
        return Response(content=resp.content, status_code=resp.status_code,
                        media_type="application/json")


@app.get("/health")
async def health():
    return {"status": "ok", "memoryweb": MW_URL, "openai": OPENAI_URL}


if __name__ == "__main__":
    uvicorn.run(app, host="127.0.0.1", port=4141, log_level="warning")
```

### Wiring clients to the proxy

| Client | How to point at proxy |
|--------|----------------------|
| Python openai SDK | `OpenAI(base_url="http://localhost:4141/v1")` |
| Cursor | Settings → OpenAI Base URL → `http://localhost:4141/v1` |
| Windsurf | Settings → Model → Custom base URL → `http://localhost:4141/v1` |
| curl / httpie | `--url http://localhost:4141/v1/chat/completions` |
| LangChain | `ChatOpenAI(openai_api_base="http://localhost:4141/v1")` |

The proxy passes the client's own `Authorization` header through, so each tool
uses its own API key — the proxy never stores or logs keys.

## Risks and Failure Modes

| Risk | Mitigation |
|------|-----------|
| MemoryWeb down when proxy starts | All MemoryWeb calls are wrapped in try/except; proxy degrades to passthrough |
| MemoryWeb returns irrelevant noise | Tune `/api/search` `limit` to 3–5; add a relevance score threshold filter |
| Injected memories push context over model limit | Cap memory injection at 2000 tokens; truncate oldest memories first |
| API key leakage via proxy logs | Proxy never logs Authorization header; only logs method + path |
| Proxy adds latency | MemoryWeb timeout is 3s hard cap; p99 latency impact is negligible |
| Port 4141 conflicts | Configurable via CLI arg; document it |
