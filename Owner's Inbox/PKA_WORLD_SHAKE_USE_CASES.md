# PKA World-Shake Use Cases

This is the shortest credible path from the current PKA stack to category-defining products.
The constraint is strict: do not break the current workspace, and do not add surface area
that weakens validation, evidence, or operator control.

## 1. Verified Agent Ops Platform

### What It Is
An enterprise operating layer for AI agents where every consequential task is routed,
executed, validated, scored, and archived with proof.

### Why It Matters
Most agent products optimize for autonomy demos. This one optimizes for trust:
- durable task lifecycle
- real validation gates
- evidence packs
- manifest-backed delivery
- observability history
- resilience testing

### Current Assets Already In Hand
- Task ledger and lifecycle controls
- Session gates
- Audit layer
- E2E validation
- Resilience testing
- Scorecard from real operational history
- Evidence pack generation
- Message protocol
- AI Army group chat coordination path
- Spark and AWS infrastructure

### First Non-Breaking Build Slice
Ship this as the default PKA operator layer:
- `python scripts/pka_operator_run.py`
- `python scripts/pka_scorecard.py`
- `python scripts/pka_observability.py`
- `python scripts/pka_evidence_pack.py --task-id TASK-...`

### Expansion Path
- standalone repo root
- install/bootstrap script
- hosted validation dashboard from generated reports
- durable runtime/event replay outside file-only orchestration

## 2. Sovereign Mission Execution Stack

### What It Is
A sovereign AI operations stack for defense, govcon, regulated healthcare, and critical
enterprise environments where cloud dependence, weak provenance, and unverified outputs
are unacceptable.

### Why It Matters
This is where your existing assets compound:
- Spark-1 / Spark-2 cluster
- AI Router Platform
- AI Army communication system
- DoD systems
- GovCon pipeline
- Medical translation
- provenance/patent work
- teacher-student learning and learning agents

### Current Assets Already In Hand
- Specialized task routing
- audit and validation controls
- cluster infrastructure
- local model and adapter ecosystem
- patent-backed innovation archive
- AWS backup and traceability surfaces

### First Non-Breaking Build Slice
Use the current workspace as the control plane and position the runtime around three
high-stakes functions:
- compliance-aware document reasoning
- validated cyber / operations triage
- provenance-backed decision support

### Expansion Path
- signed evidence chain for outputs
- retrieval audit logging
- isolated deployment profile
- policy packs per regulated domain

## Rules For Both Paths

Do not compromise these:
- green means proven
- no self-scoring from setup files
- no consequence-bearing delivery without evidence
- no product surface that outruns operational truth
- no new runtime layer unless it improves durability, observability, or adoption
