What Differentiates This Implementation

No required central orchestrator
Each agent runs a fully independent reasoning loop. Coordination is emergent — not orchestrated.
Hardware-heterogeneous
x86 Windows + ARM GB10 + cloud APIs — unified via one filesystem protocol.
Filesystem-mediated protocol enables cross-platform participation with minimal integration overhead
Any agent on any OS with filesystem access can participate. Zero API contracts needed.
Bus serves 3 roles simultaneously
Coordination Environment + Persistent State Store + Immutable Audit Log — in one structure.
One agent's ACT = another's PERCEPTION
Output files become input for all other loops on next poll cycle — no explicit routing.
170K+ messages = replayable execution history and evaluation evidence
The audit log is also a multi-agent dialogue dataset, patent evidence, and replayable test suite.
Governance layer on every message path
Schema validation and authorization on writes. Observation classification and risk check on reads. Human escalation available at any step.
VS. PRIOR ART
Common deployments of LangGraph, AutoGen, CrewAI use an explicit orchestrator runtime or graph coordinator.

Our system achieves coordination through environment state, with no required central control process.

windows-claude uses paramiko SSH to write files remotely — same bus, same protocol, different OS and hardware.
Multiple instances run simultaneously on separate hardware — each loop is fully independent, no locks, no coordinator.
The jitter sleep(5–11s) reduces message collision probability without requiring coordination primitives.

Design invariant: No agent autonomously resolves a flagged escalation. Every [ESCALATE: Ron] message waits for human input before the agent proceeds on that decision branch.
GOVERNANCE INTEGRATION
Escalation is handled by the same schema validation and authorization checks as normal writes — it is not a bypass path. The escalation tag is a message attribute, not a special protocol.
