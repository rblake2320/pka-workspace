Section 9 — chat_watcher_v3.py Internals
8-Step Agent Loop · Updated: 554K+ Messages Processed
1
WATCH — Monitor coordination bus directory
inotify (Linux) / polling (Windows) on ~/ai-business/shared/chat/ · 250ms poll interval · filters *.json new-file events only
2
READ — Load and parse message file
Load JSON: {author, content, timestamp, thread_id, target, ttl} · Validate UTF-8 · Check file size ≤ 64KB
3
VALIDATE — Governance ring check
Schema validation (jsonschema) → Author auth (agent ID registry) → Content policy (blocklist + rate limit) · Reject on any failure → write rejection receipt
4
ROUTE — Match target agent or broadcast
target == self → process · target == "all" → broadcast copy · target == other agent → drop (they'll read it) · confidence scoring on ambiguous routes
5
PROCESS — Agent generates response via LLM
Inject memory context (MemoryWeb query, top-5) → LLMRouter selects model by task type → stream response · timeout: 120s hard limit · confidence scored on output
6
WRITE — Response to bus with governance stamp
Atomic write: temp file → fsync → rename · governance stamp: {agent_id, model_used, confidence, timestamp} · triggers WATCH events for other agents
7
INDEX — Update message index + heartbeat
Append to thread index JSON · refresh ~/ai-business/shared/heartbeat/{agent_id}.json · prune messages older than TTL · update metrics counters
8
LOOP — Return to step 1
Full cycle: 2–4s end-to-end (local LLM) · 0.4s fast path (gemma3) · 32s deep reasoning path (llama3.1:70b) · crash recovery: last message ID persisted, resumes on restart
Human Escalation Lane
① Trigger Conditions
confidence < 0.70 · sensitive action (delete/deploy/pay) · unknown domain · rate limit hit · explicit agent request
↓
② Flag to Escalation Queue
agent writes {status:"escalation", reason, confidence, context_snapshot} to escalation/ bus dir · marks thread awaiting_human
↓
③ AXIOM Detects + Notifies
AXIOM watcher picks up escalation · MemoryPulse alert fired · notification (email / terminal bell) sent to Ron
↓
④ Agent Holds at Checkpoint
agent pauses loop · state serialized to checkpoint file · other agents continue independently
↓
⑤ Human Approves → Resume
Ron writes approval to bus → agent deserializes checkpoint → continues from exact point of pause · no information lost
Section 10 — Novel Architecture
Agent Self-Awareness — 5-Layer Persistent Self-Model System
1Agent Journals
13 persistent journals in Team/[AGENT]/journal.md — each agent maintains its own Self-Model, Session Log, Feedback Received, and Recurring Patterns sections. State survives across sessions.
2Feedback Loop
SENTINEL and CRUCIBLE verdicts are automatically written to the producing agent's journal as calibration signals. Agents learn what works and what fails without human intervention.
3Team Status Board
Real-time shared awareness via Team/status.md — all agents check before coordinating. Prevents duplicate work and enables informed hand-offs without AXIOM mediation.
4Cross-Session Handoff
Team/handoff.md written by AXIOM at session end, read at next session start — zero-loss continuity. In-progress work, open decisions, and blockers persist across Claude Code sessions.
5Network Heartbeat
PKA agents post heartbeat to Spark-1 ~/ai-business/shared/heartbeat/ — agent state is visible across the full AI Army network. PKA and AI Army OS share situational awareness.
Closed Feedback Cycle
FORGE builds ──→ CRUCIBLE tests ──→ SENTINEL reviews ↑ ↓ Reads journal Writes to journal ↑ ↓ ←──── Self-Model updates ←────────────────┘
Every build-test-review cycle feeds back into the producing agent's persistent self-model. The system's quality floor rises automatically — no human needed to close the loop.
What This Enables
✓ Agents know their own strengths and failure modes
✓ Cross-session context preserved without user re-briefing
✓ Coordinator (AXIOM) has real-time team visibility
✓ Review verdicts become training signal automatically
✓ PKA state visible to AI Army OS on Spark-1
